# 정산업무 프로세스 가이드

PG 결제 시스템의 정산업무 전체 프로세스를 설명합니다.

## 1. 정산 주기

정산은 가맹점 계약에 따라 다음 주기로 진행됩니다:

| 정산 주기 | 설명 | 적용 대상 |
|----------|------|----------|
| D+0 | 당일 정산 | VIP 가맹점 |
| D+1 | 익일 정산 | 일반 가맹점 (기본) |
| D+2 | 2영업일 후 정산 | 신규 가맹점 |
| WEEKLY | 주간 정산 (매주 월요일) | 소액 거래 가맹점 |
| MONTHLY | 월간 정산 (매월 1일) | 특수 계약 가맹점 |

## 2. 정산 프로세스 단계

### 2.1 정산 대상 집계 (매일 00:00~01:00)

1. 전일 `DONE` 상태의 결제 건 집계
2. 전일 `CANCELED`, `PARTIAL_CANCELED` 건 차감
3. 가맹점별 수수료 계산
4. 정산 예정 금액 산출

### 2.2 정산 검증 (01:00~02:00)

1. PG사 원장과 대사
2. 불일치 건 자동 플래그
3. 관리자 확인 대기 건 분류

### 2.3 정산 확정 (09:00)

1. 검증 완료 건 정산 확정
2. Settlement 레코드 생성 (`status: PENDING`)
3. 가맹점 알림 발송

### 2.4 지급 실행 (14:00)

1. 정산 확정 건 은행 이체 요청
2. 이체 결과 수신 (보통 30분 내)
3. 성공 시 `status: COMPLETED`로 변경
4. 실패 시 `status: FAILED`로 변경, 재시도 큐 등록

## 3. 정산 금액 계산 공식

```
정산금액 = 결제금액 - 환불금액 - 수수료
수수료 = (결제금액 - 환불금액) × 수수료율
```

### 결제 수단별 수수료율

| 결제 수단 | 수수료율 | 비고 |
|----------|---------|------|
| 신용카드 | 3.3% | 일반 가맹점 기준 |
| 체크카드 | 2.5% | 신용카드보다 저렴 |
| 가상계좌 | 300원/건 | 건당 고정 수수료 |
| 간편결제 | 3.3% | 카카오페이, 네이버페이 등 |

## 4. 정산 상태 (Settlement Status)

- **PENDING**: 정산 대기 (검증 완료, 지급 대기)
- **PROCESSING**: 이체 처리 중
- **COMPLETED**: 지급 완료
- **FAILED**: 지급 실패 (재시도 필요)

## 5. 정산 조회 방법

### 5.1 가맹점별 정산 현황
- "어제 A가맹점 정산 내역 보여줘"
- "이번 달 정산 완료된 건 조회"

### 5.2 정산 상태별 조회
- "PENDING 상태인 정산 건 조회"
- "FAILED 정산 건 목록"

## 6. 주의사항

1. **정산 마감 시간**: 매일 23:59:59까지의 거래가 당일 정산 대상
2. **환불 반영**: 환불 발생 시 다음 정산에서 차감
3. **공휴일 처리**: 공휴일 다음 영업일에 합산 정산
4. **최소 정산 금액**: 1,000원 미만은 이월
